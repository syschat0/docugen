# DocuGen 개선 로드맵

다양한 유형의 글 지원을 목표로 한 우선순위별 진행목록.
(2026-07-08 분석 기준. M1은 범위 확정, M2 이후는 착수 시 세부 설계.)

---

## M1. 참고문헌 표기법 2종 지원 — 확정, 즉시 착수 가능

소스가 사실상 100% URL 형식이므로 메타데이터 추출 없이 렌더링 분기만으로 구현.
다른 마일스톤에 의존하지 않는 독립 작업.

- [x] 1-1. `citation_style` 프로젝트 설정 추가 (`numeric` | `author_date`, 기본 `numeric`)
      — `app/schemas/projects.py`, `app/db/repositories.py`(get/set_project_settings), `app/api/settings.py`
- [x] 1-2. 번호식 렌더러 정비: 현행 `[[n]](url)` + 참고문헌 목록에 사이트명·접속일 추가
      — `app/services/citations.py` (접속일 = research 아티팩트 `updated_at`)
- [x] 1-3. 저자-연도식 렌더러: 본문 `(사이트명, n.d.)` 링크 + 가나다/알파벳순 목록
      — 동일 도메인 다건 시 `n.d.-a`, `n.d.-b` 접미사 구분, `www.` 제거, `file://` 참조는 `[사용자 제공 자료]`로 표기
- [x] 1-4. 이음새 가드 확장: `global_citation_numbers()` 정규식이 저자-연도 링크 패턴도 인식
      — `app/services/citations.py:53`, `smooth_chapter_seams` 경로
- [x] 1-5. `final_merge`에 스타일 주입 + 스타일 변경 시 캐시 유지한 채 merge만 재실행
      — `app/db/repositories.py` (`_local_merge`, draft 재생성 조건)
- [x] 1-6. 설정 UI에 표기법 토글 추가
      — `app/static/index.html`, `app/static/app.js` (기존 프로젝트 설정 패널)
- [x] 1-7. 테스트: 두 스타일 렌더링, n.d. 접미사 규칙, file 참조, 이음새 가드
      — `tests/test_citations.py` 확장
- [x] 1-8. (선택 후속) 메타태그·JSON-LD 파싱으로 저자/게시일 있는 소스 승격
      — `(사이트명, n.d.)` → `(저자, 연도)`. `app/services/page_meta.py` 신설,
      browser/HTTP/챕터 리서치 fetch 경로 연결 (사용자 URL 참조는 DB 스키마
      변경이 필요해 제외 — 도메인 폴백 유지)

## M2. 문서 유형(장르) 기반 마련 — 유형 다양화의 토대

- [x] 2-1. `document_type` 필드 추가 (프로젝트 생성/수정 스키마·DB·API)
      — `session.py` 컬럼 마이그레이션 헬퍼, `GET /doc-types` 엔드포인트
- [x] 2-2. 유형 프로파일 레지스트리: `app/services/doc_types.py`
      (1차 유형 6종: 보고서(현행) / 소논문(학교 과제) / 블로그·칼럼 / 에세이
      / 기술 문서 / 발표 대본 — 리서치 기본값, 인용 on/off, 제목 번호,
      기본 분량, 문체, 단계별 프롬프트 가이드 선언)
- [x] 2-3. 인테이크에서 유형 자동 분류(best-effort) + UI에서 확인·변경
      — 유형 미지정 시 실행 시작에 분류 후 저장, 변경 시 파이프라인 무효화
- [x] 2-4. 프로젝트 생성 UI 유형 선택(자동 감지 포함) + 생성 조건 패널에서 변경
      — M2에서 함께 배선된 것: brief/outline/섹션 프롬프트 유형 가이드 주입,
      인용 없는 유형(에세이·발표 대본)의 마커 제거·Sources 생략,
      제목 번호 유형별 적용, 에세이 리서치 기본 off

## M4. 구조·분량 규칙 유형별화 — M2 의존

- [x] 4-1. 제목 번호 매김을 유형별 옵션으로 — M2에서 `numbered_headings`로 반영.
      하위 제목 금지 규칙은 계층 아웃라인 트리가 구조 분할을 담당하므로
      전 유형 공통 유지로 판단 (유형별 옵션 불필요)
- [x] 4-2. 문서 전체 목표 분량 → 섹션별 배분
      — 우선순위: 설정 패널 `target_length` > brief가 요청문에서 추출한
      `target_length_chars` > 없음. 배분은 실행 시 비례 재계산(저장 안 함,
      섹션당 150~3,000자 클램프), outline/챕터 확장 프롬프트에도 예산 전달.
      normalize의 500자 하드코딩이 프로파일 기본 분량을 가리던 문제 수정

## M5. 문체·인테이크 심화 — M2 의존

      — M2에서 인테이크 프롬프트에 유형 관례·질문 가이드 주입으로 부분 반영
- [x] 5-2. 스타일 카드: 어체·어조·시점·시제·문장 리듬·어휘 + 퓨샷 예문을

- [x] 5-3. 샘플 글 업로드 → 스타일 카드 추출 — references를 `kind="style"`로
